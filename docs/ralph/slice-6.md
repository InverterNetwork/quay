## Goal

Atomic claims with an ownership fence; tick auto-releases timed-out
claims; repeated expirations park the task in `orchestrator_loop`.
`submit-brief` schedules a pending attempt and clears the claim but
never spawns directly. `escalate-human` is a claim-scoped SQL +
artifact transition only — Slack posting is deferred to Slice 8.

## Red tests (must exist with these names and must pass)

| Test name | Proves |
|---|---|
| `test_015_concurrent_claims_only_one_wins` | Concurrent claims on the same `awaiting-next-brief` task arbitrate atomically; one caller gets a fresh `claim_id`, others fail. |
| `test_016_claim_timeout_auto_releases` | Tick auto-releases stale claims by clearing `claim_id`, moving the task back to `awaiting-next-brief`, logging `claim_expired`, and incrementing the expiration counter. |

Place tests under `tests/claim/` and `tests/escalate/`.

## Minimal implementation

- `claim_task`, `release_claim`, `submit_brief`, `escalate_human`
  service entry points with SQL predicates that fence on `claim_id`.
- Claim-timeout sweep inside `tick_once`.
- Error taxonomy: `claim_lost`, `cancelled`, `wrong_state`,
  `unknown_task`. Use a discriminated union; do not throw raw
  errors at boundaries.
- Claim-expiration counter + cap → `orchestrator_loop`.
- Escalation artifact must include a stable per-escalation `nonce`
  generated by the deterministic ID generator so Slice 8's recovery
  can find it.

## Done criteria

- A stale claimant cannot mutate a re-claimed task.
- Claim expirations counted and reset exactly as specified.
- Claim-scoped writes distinguish cancellation from claim loss.
- All eight tests pass; suite green; typecheck green.
